Labour Management Manual

When you first open the database you will be presented with this form which links to all the functions the database provides.

Create New Pirate: Add a new pirate to your database
Manage Pirates: Manage the details of each pirate such as their skills and what shops they have jobs at
New Shop: Add a new shop to the database
Manage Shop: View the details of labour output in each shop and the status of the alts employed there

Most of this form is fairly straight forward and doesn't need any explanation.
Most alts in my experience only have one skill type but more can be added later via the Manage Pirates form.

Again a very simple form with no explanation needed.

This form is where you manage all the information on each individual Pirate.
On the left is a list of all the pirates currently in the database, just click on one to bring up the information for it.
To appear in a shop's list of employees an alt must take a job, to do that simply choose the shop from the dropdown box in the Take new job area and choose take job. To quit do the same in the quit job box. (Known Bug: Unable to quit first job, if you really need to do this you must delete the entry via the tables)
If the pirate has any jobs or skills the information will be shown in the boxes in the top right. To edit a pirates skills enter the information into the add/update skill box.

This is the screen that you will come to most often to check what alts are about to undormant.

The labour output displays the maximum output your alts will send to this shop - Note that this is a maximum, if your alts have several jobs then this figure will be incorrect (realistically aswell alts only provide 90% of what they are capable in a single shop)

The dormanting workers list will display any workers that are within a day or two of undormanting IN ANY SHOP.

The active workers list displays all active workers that are in the currently selected shop

To update a worker FIRST select the relevant shop type, it doesn't really matter which one as long as it is the same type. If you don't do this you will recieve an EOF error as the database will search for a skill that does not exist or it will change the wrong skill. After this it is a simple case of entering the pirate name and selecting one of the options below (Skill level is only needed if you are updating it)

Known Bugs: If an alt has several jobs it will appear several times in the Dormanting Workers list
